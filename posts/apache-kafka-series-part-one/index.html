What is Log?#

Very first line of the Kafka paper[2] says

Log processing has become a critical component of the data pipeline for consumer internet companies

What is log data here that they are talking about? First thing it came in mind was application logs which kind a looks like this,

These Tomcat logs are humanly readable which simplest form of log that they are talking about which software produces.

Log is a chronological record of actions/performance generated by system or human.

Here, system can be a simple web server, operating system or software which generates syslogs[3], most database systems have transaction logs etc. System puts these logs into single log file which we know as Logging. Some logs might be human redeable text file or some can be binary which Database system provides.

Transaction logs which is generated by database is a chronological history of record of change made to the database. Transaction logs are separated from actual data files. In MySQL world it is known as ‘Binary Log’ and in Postgres, it is known as ‘Write-Ahead Log (WAL)’.
